description: Build pitch decks, mood boards, and creative briefs with embedded AI visuals as HTML, PDF-ready, or Markdown
---

Run the design-docs skill. Capture the doc brief (type, audience, brand context), run `gflow doctor`, generate section illustrations with `gflow image` (inheriting `--project <brand>` if a brand exists), build the document with a clear typographic hierarchy on a 12-column grid with generous py-24+ spacing, apply the brand system if a brand-identity run exists, output self-contained HTML or Markdown plus an assets folder, run the copy audit to ban filler verbs, em-dashes, and generic claims, and run the full quality gate before delivering. Execute every gflow command with one shared `--project <name>` for the run (the brand project if one exists) and `--no-headed`; only `gflow auth login` runs headed.
